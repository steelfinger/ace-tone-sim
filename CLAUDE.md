# Ace Tone FR-1 Simulator — Claude Guidelines

## Project overview

React Native (Expo) simulator of the Ace Tone FR-1 Rhythm Ace (c. 1967), one of the earliest consumer drum machines. All drum sounds are synthesized via the Web Audio API (`react-native-audio-api`) — no samples.

## Stack

- **Expo SDK 55** / React Native 0.83
- **react-native-audio-api 0.11.7** — native Web Audio API port (sample-accurate scheduling)
- **react-native-reanimated 4** + **react-native-gesture-handler 2** — rotary knobs
- **Zustand 5** — global sequencer state
- **TypeScript strict**

## Architecture

```
src/
  audio/
    context.ts     singleton AudioContext + shared noise AudioBuffer
    voices.ts      8 synthesized voices; export trigger: Record<VoiceId, fn>
    bus.ts         per-voice GainNode for cancel/mute routing
    scheduler.ts   lookahead clock + combine() OR-reduces selected patterns
  patterns/
    index.ts       16 factory presets as { id, name, steps, grid }
  state/
    sequencer.ts   Zustand store — bpm, selected[], mutes, running, currentStep
  ui/
    Panel.tsx      main screen — rhythm grid, cancel row, knob bank, start/stop
    Knob.tsx       pan-gesture rotary knob (Reanimated)
```

## Key design decisions

- **Synthesized voices, not samples** — each voice is a small Web Audio graph (oscillators + filters + gain envelopes). Tuned to the FR-1's transistor-based AR character.
- **Lookahead scheduler** — JS timer fires every 25 ms, schedules audio 100 ms ahead using `ctx.currentTime`. Never trigger voices directly from `setInterval`.
- **Pattern combining** — selecting multiple rhythm buttons OR-combines their grids (faithful to the original diode matrix). Waltz (12-step, 3/4) can only combine with other Waltz patterns.
- **Cancel buttons** — the 4 cancel buttons (Bass, Cymbal, Claves, Cowbell) mute the voice via `bus.ts`; they do not manually trigger hits.

## Known issues / caveats

- **Pattern accuracy** — the 16 preset grids are transcribed by ear from demos. Treat as a starting point; refine against reference recordings.
- **ios/ and android/** are gitignored (generated by `npx expo prebuild`).

## Dev workflow

```bash
npm install
npx expo prebuild          # generates ios/ and android/
npx expo run:ios           # build + launch simulator
```

After any dependency change that affects native code, re-run `expo prebuild`.
